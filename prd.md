# 智能策略SaaS平台 - 产品需求文档 (PRD)

## 一、文档信息

| 项目 | 内容 |
|-----|------|
| 产品名称 | 智能策略SaaS平台 |
| 版本 | v1.0 MVP |
| 目标用户 | 企业运营团队、增长团队 |
| 编写日期 | 2026-03-25 |

---

## 二、需求背景

### 2.1 痛点分析

| 痛点 | 现状 | 影响 |
|-----|------|------|
| 策略设计门槛高 | 依赖数据分析+策略经验 | 新手运营难以产出有效策略 |
| 执行效率低 | 跨系统操作，手动投放 | 策略上线周期长（3-5天） |
| 效果评估难 | 缺乏AB实验能力 | 无法确定策略真实效果 |
| 优化迭代慢 | 人工回收数据，手动调整 | 策略迭代周期以周为单位 |

### 2.2 目标用户画像

**主要用户：运营经理**
- 负责用户增长/留存运营
- 需要定期设计并执行运营策略
- 希望有数据洞察辅助决策

**次要用户：数据分析师**
- 协助运营分析人群数据
- 验证策略效果

## 三、功能需求

### 3.1 战略人群管理

#### 3.1.1 人群定义（人工定义1/2/3级）

**需求描述**：
- 支持创建最多3个战略人群
- 每个人群通过标签圈选定义
- 支持人群优先级标记（1级最高）

**功能细节**：

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 人群名称 | 文本 | 是 | 如"高价值流失风险用户" |
| 战略等级 | 单选 | 是 | 1/2/3级 |
| 标签条件 | 条件组 | 是 | 支持AND/OR组合 |
| 人群说明 | 文本 | 否 | 业务定义说明 |
| 数据刷新周期 | 单选 | 是 | 实时/T+1/T+7 |

**标签条件示例**：
```
(用户价值等级 = "高价值") AND (最近购买时间 > 30天) AND (最近访问时间 <= 7天)
```

#### 3.1.2 人群看板

**需求描述**：
- 展示3个人群的核心指标
- 支持历史趋势对比
- AI智能诊断报告

**展示指标**：
| 指标 | 说明 |
|-----|------|
| 人群规模 | 当前覆盖用户数 |
| 日均活跃 | 近7天平均DAU |
| 转化率 | 近30天目标转化率 |
| GMV贡献 | 近30天GMV占比 |
| 流失风险 | AI评估的流失风险等级 |

**AI诊断报告内容**：
- 当前状态概述（健康/预警/危险）
- 发现的机会点（1-3条）
- 潜在风险（1-3条）
- 建议采取的策略方向

---

### 3.2 策略编排

#### 3.2.1 策略画布

**需求描述**：
可视化编排策略五要素：人群→触达→承接→权益→商品

**画布元素**：

```
┌────────────────────────────────────────┐
│  [人群选择器] ──→ [触达配置]           │
│       │                                  │
│       ↓                                  │
│  [承接页] ←── [权益配置] ←── [商品]     │
└────────────────────────────────────────┘
```

**各元素配置详情**：

**1. 人群选择器**
- 选择战略人群（单选）
- 显示人群预估规模
- 支持添加额外筛选条件（可选）

**2. 触达配置**
- 触达渠道（多选）：
  - 外部：Push、SMS、微信公众号
  - 端内：弹窗、Banner、站内信
- 触达顺序：并行/串行
- 触达时机：立即/定时/周期性
- 频次控制：用户级每日/每周上限

**3. 承接配置**
- 承接类型：H5页面/小程序/原生页/弹窗
- 页面搭建：
  - 支持选择模板
  - 支持上传素材
  - 支持配置跳转链接

**4. 权益配置**
- 权益类型：优惠券/积分/会员卡/无
- 优惠券：
  - 选择已有券模板
  - 或创建新券
  - 设置发放数量
- 发放时机：立即/满足条件（如首次点击）

**5. 商品配置**
- 商品选择方式：
  - 指定商品（输入商品ID）
  - 商品池（选择商品池）
  - 智能推荐（算法推荐）
- 展示数量限制

#### 3.2.2 策略保存与发布

**策略状态流转**：
```
草稿 → 测试中 → 运行中 → 已暂停 → 已归档
  ↓       ↓         ↓
可编辑   AB实验中   监控中
```

**发布检查项**：
- [ ] 人群规模>0
- [ ] 至少配置1个触达渠道
- [ ] 权益库存充足（如配置了权益）
- [ ] 页面素材审核通过（如有）

---

### 3.3 AB实验

#### 3.3.1 实验配置

**实验参数**：
| 参数 | 说明 | 默认值 |
|-----|------|-------|
| 实验流量 | 参与实验的用户比例 | 10% |
| 对照组比例 | 对照组占实验流量的比例 | 50% |
| 实验时长 | 实验运行天数 | 3天 |
| 胜出标准 | 判断胜出的指标 | 转化率 |
| 最小提升 | 认为有效的最小提升 | 10% |
| 置信度要求 | 统计显著性要求 | 95% |

#### 3.3.2 实验报告

**报告内容**：
- 实验组vs对照组核心指标对比
- 统计显著性检验结果
- 实验结论（胜出/未胜出/继续观察）
- 建议操作（全量/停止/延长实验）

---

### 3.4 效果回收

#### 3.4.1 实时数据看板

**数据维度**：
| 维度 | 指标 |
|-----|------|
| 触达 | 发送数、送达数、送达率 |
| 点击 | 点击数、点击率 |
| 转化 | 转化数、转化率、GMV |
| 成本 | 权益成本、单客成本、ROI |

**更新频率**：5分钟刷新

#### 3.4.2 自动迭代规则

**自动全量条件**（需同时满足）：
- AB实验显著性 > 95%
- 实验提升 > 10%
- 实验运行时长 >= 3天

**自动下线条件**（满足任一）：
- 送达率 < 50%
- 点击率 < 1%
- ROI < 0.5

## 四、非功能需求

### 4.1 性能需求

| 场景 | 要求 |
|-----|------|
| 页面加载 | 首屏 < 2s |
| 人群预估 | < 3s |
| 策略发布 | < 5s |
| 数据看板 | < 3s |
| 批量推送 | 支持100万用户/小时 |

### 4.2 安全需求

- 人群数据脱敏展示
- 操作日志记录

### 4.3 可用性需求

- SLA: 99.9%
- 数据备份：每日全量备份
- 故障恢复：RTO < 30分钟

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|-----|------|---------|
| 人群API延迟高 | 策略发布慢 | 异步化人群计算，支持队列处理 |
| 触达渠道故障 | 策略执行失败 | 多渠道兜底，失败重试机制 |
| AI建议不准确 | 运营不信任 | 标注AI建议仅供参考，保留人工决策权 |
| 数据权限复杂 | 开发周期长 | 前期简化权限，后期迭代完善 |
