# 智能策略平台 MVP PRD

## 一、项目概述

### 1.1 产品定位
AI原生策略运营平台，通过自然语言对话，帮助运营快速创建人群策略并自动执行。

### 1.2 MVP范围
- **数据源**：仅接入1个人群DB API（数据中台）
- **核心能力**：人群诊断 → AI策略生成 → 策略执行 → 效果回收
- **AI能力**：Copilot对话式交互、智能策略生成、效果分析

### 1.3 目标用户
企业运营人员，负责用户增长/留存策略的制定与执行。

## 二、功能范围

### 2.1 功能清单

| 模块 | 功能 | 说明 | 优先级 |
|-----|------|------|-------|
| 人群管理 | 战略人群定义 | 人工定义3个战略人群（标签圈选） | P0 |
| 人群管理 | 人群数据看板 | 展示规模/转化/趋势 | P0 |
| 人群管理 | AI智能诊断 | AI分析人群健康度，发现问题/机会 | P0 |
| 策略编排 | AI策略生成 | 自然语言生成策略五要素 | P0 |
| 策略编排 | 策略画布 | 可视化编辑策略 | P0 |
| 策略编排 | 策略保存/发布 | 草稿/运行状态管理 | P0 |
| 策略执行 | 人群包获取 | 从人群DB API拉取人群 | P0 |
| 策略执行 | 触达下发 | Push + 短信（先实现Push） | P0 |
| 策略执行 | AB实验 | 实验组/对照组分流 | P1 |
| 效果回收 | 实时数据看板 | 发送/点击/转化数据 | P0 |
| 效果回收 | AI效果分析 | AI分析策略效果，给出优化建议 | P0 |

### 2.2 不在MVP范围
- 商品推荐系统
- 复杂的端内承接页搭建
- 多租户权限管理
- 策略自动迭代（全自动）
- 连接器市场（二期）

## 六、数据模型（简化版）

### 6.1 核心表结构

```sql
-- 战略人群表
CREATE TABLE strategic_segments (
    id VARCHAR(64) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    level TINYINT NOT NULL COMMENT '1/2/3级',
    definition JSON NOT NULL COMMENT '标签条件',
    health_score INT COMMENT '健康度评分',
    ai_diagnosis JSON COMMENT 'AI诊断结果',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 策略表
CREATE TABLE strategies (
    id VARCHAR(64) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    status VARCHAR(32) DEFAULT 'draft',
    segment_id VARCHAR(64) NOT NULL,
    components JSON NOT NULL COMMENT '五要素配置',
    ai_generated BOOLEAN DEFAULT FALSE,
    ai_prompt TEXT,
    prediction JSON COMMENT '效果预测',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 策略执行表
CREATE TABLE strategy_executions (
    id VARCHAR(64) PRIMARY KEY,
    strategy_id VARCHAR(64) NOT NULL,
    status VARCHAR(32) NOT NULL,
    segment_size INT,
    channel_results JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- AB实验表
CREATE TABLE ab_tests (
    id VARCHAR(64) PRIMARY KEY,
    strategy_id VARCHAR(64) NOT NULL,
    traffic_split DECIMAL(3,2) DEFAULT 0.10,
    status VARCHAR(32) DEFAULT 'pending',
    results JSON,
    winner_variant VARCHAR(32),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Agent记忆表（用于AI学习）
CREATE TABLE agent_memories (
    id VARCHAR(64) PRIMARY KEY,
    agent_type VARCHAR(32) NOT NULL,
    content TEXT NOT NULL,
    embedding VECTOR(1536),  -- 向量化存储
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 九、验收标准

### 9.1 功能验收

| 功能 | 验收标准 |
|-----|---------|
| 人群管理 | 可创建3个人群，标签圈选正常 |
| AI诊断 | 诊断报告包含评分/问题/建议，生成时间<5s |
| AI策略生成 | 支持自然语言输入，生成完整策略，置信度>80% |
| 策略执行 | 发布后10分钟内开始触达 |
| 效果回收 | 数据看板5分钟更新，指标准确 |
| AB实验 | 分流正确，报告数据准确 |

### 9.2 性能指标

- AI响应时间 < 3s
- 策略发布到执行 < 5分钟
- 人群圈选 < 30s（10万用户）
- 页面加载 < 2s

## 十、风险与应对

| 风险 | 应对 |
|-----|------|
| 人群DB API延迟高 | 异步圈选，支持轮询获取结果 |
| AI生成策略不准确 | 提供人工调整入口，标注仅供参考 |
| Push渠道故障 | 记录失败原因，支持重试 |
| 数据权限问题 | MVP暂不考虑复杂权限，后期迭代 |
