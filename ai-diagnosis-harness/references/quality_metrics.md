# 诊断质量指标

## 核心质量指标

### 1. 结构合规率 (Structure Compliance)
```
结构合规率 = 通过Schema验证的诊断数 / 总诊断数 × 100%

目标: ≥ 99%
临界: < 95% 需立即修复
```

### 2. 数值合理率 (Value Rationality)
```
数值合理率 = 数值在合理范围内的诊断数 / 总诊断数 × 100%

合理范围定义:
- 健康度: 0-100
- 流失率: 0-50% (超过50%视为异常)
- 转化率: 0-30% (超过30%需验证)
- 人群规模: 与基础数据误差 < 1%

目标: ≥ 98%
```

### 3. 逻辑一致率 (Logical Consistency)
```
逻辑一致率 = 无逻辑冲突的诊断数 / 总诊断数 × 100%

检查项:
- 健康度与问题严重程度匹配
- 变化趋势方向正确
- 影响人数 ≤ 人群规模
- 机会与问题不冲突

目标: ≥ 95%
```

### 4. 时效合规率 (Timeliness)
```
时效合规率 = 数据时效性合格的诊断数 / 总诊断数 × 100%

合格标准: 诊断基于24小时内的数据

目标: 100%
```

## 深度质量指标

### 5. 人工一致率 (Human Agreement)
```
人工一致率 = 人工判定与AI一致的诊断数 / 抽样诊断数 × 100%

采样方法:
- 每人群每月至少1次人工复核
- 关键人群 (P0) 每周复核
- 异常诊断 (health_score < 50) 强制复核

目标: ≥ 85%
```

### 6. 预测准确率 (Prediction Accuracy)
```
预测准确率 = 预测与实际结果一致的策略数 / 有预测的策略数 × 100%

评估周期: 策略执行后7天、30天

目标: ≥ 70% (置信度 > 80% 的预测)
```

### 7. 建议采纳率 (Adoption Rate)
```
建议采纳率 = 被采纳的建议数 / 总建议数 × 100%

反映建议的实用性和可操作性

目标: ≥ 60%
```

## 质量等级

| 等级 | 综合得分 | 说明 |
|-----|---------|------|
| A (优秀) | ≥ 95 | 质量稳定，可全量发布 |
| B (良好) | 85-94 | 质量良好，需关注边缘case |
| C (及格) | 70-84 | 存在问题，需优化后发布 |
| D (不合格) | < 70 | 严重问题，禁止发布 |

综合得分 = 结构合规率×0.2 + 数值合理率×0.2 + 逻辑一致率×0.25 + 人工一致率×0.35
