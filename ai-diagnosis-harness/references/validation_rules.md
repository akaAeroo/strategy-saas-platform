# 诊断结果验证规则

## 1. 数值范围规则

### 健康度评分
- 范围: 0-100 整数
- 等级映射:
  - 90-100: excellent (优秀)
  - 70-89: good (良好)
  - 50-69: warning (需关注)
  - 0-49: critical (严重)

### 百分比指标
- 流失率: 0-100%
- 转化率: 0-100%
- 点击率: 0-100%
- 置信度: 0-100%

### 人群规模
- 必须与基础数据一致（误差 < 1%）
- 影响人数 ≤ 人群规模

## 2. 逻辑一致性规则

### 健康度与问题数量
- health_score < 50 时，必须有 critical 级别问题
- health_score > 80 时，不应有 critical 级别问题

### 变化趋势
- 流失率上升 → 健康度下降 (负相关)
- 转化率上升 → 健康度上升 (正相关)

### 问题与建议对应
- 每个 critical 问题应有对应建议
- 建议的 priority 应与问题 severity 匹配

## 3. 时间一致性

### 诊断时效性
- 诊断数据应在 24 小时内
- 环比对比周期应一致（都是7天或都是30天）

### 趋势合理性
- 短期趋势 (7天) 与长期趋势 (30天) 方向一致或差异可解释

## 4. 业务逻辑

### 人群定义一致性
- 高价值用户转化率应高于整体平均水平
- 新用户流失率可高于老用户

### 机会与问题不冲突
- 同一人群不能同时是"高风险流失"和"高增长潜力"
- 除非有明确的分群说明

## 5. 输出格式

### 必需字段
- segment_id, segment_name
- health_score, health_level
- problems[], opportunities[], suggestions[]
- generated_at (ISO 8601格式)

### 字段类型
- 数值字段必须为 number/integer，不能用 string
- 日期必须为 ISO 8601 格式
- 枚举值必须在定义范围内
